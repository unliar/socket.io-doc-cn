#Client API

IO

io在单独构建时被当作全局变量, 或者可以使用 require('socket.io-client')来引入.

#io.protocol

  • (Number)

协议的版本号.

#io(url[, options])

  • url (String)
  • options (Object)
  • Returns Socket

为传入的url创建一个新的 Manager, 并且为后来的调用尝试重用已经存在的Manager , 除非 multiplex 选项被传入了 false. 传入'force new connection': true 或 forceNew: true 等价与multiplex: false.

根据你指定的url路径创建一个新的 Socket 实例, 默认 /. 例如, 如果 url 是 http://localhost/users, 一个传输通道会被建立在 http://localhost Socket.IO 连接在 /users.

See new Manager(url[, options]) for available options.

Manager

#new Manager(url[, options])

  • url (String)
  • options (Object)
    • path (String) 被捕获的路径 (/socket.io)
    • reconnection (Boolean) 是否自动重连 (true)
    • reconnectionAttempts (Number) 重连次数 ( Infinity)
    • reconnectionDelay (Number)
      重连间隔( 1000). 会上下浮动 +/- randomizationFactor,
      实际的延迟会在 500 到 1500ms上下.
    • reconnectionDelayMax (Number) 最大连接等待时间
      ( 5000). 每次尝试会增加连接2倍的等待时间
      会有一些上下浮动
    • randomizationFactor (Number) (0.5), 0 <= randomizationFactor <= 1
    • timeout (Number) connect_error
      and connect_timeout 事件被激活的延迟时间 (20000)
    • autoConnect (Boolean) 设置值为false,你只能手动调用manager.open
  • Returns Manager

engine.io Server 创建时也有同样的选项 Socket. 请查看 options here.

#manager.reconnection([value])

  • value (Boolean)
  • Returns Manager|Boolean

设置 reconnection , 或者参数为空则返回reconnection的值 .

#manager.reconnectionAttempts([value])

  • value (Number)
  • Returns Manager|Number

设置 reconnectionAttempts , 参数为空则返回值.

#manager.reconnectionDelay([value])

  • value (Number)
  • Returns Manager|Number

设置reconnectionDelay , 参数为空则返回值.

#manager.reconnectionDelayMax([value])

  • value (Number)
  • Returns Manager|Number

设置 reconnectionDelayMax , 参数为空则返回值.

#manager.timeout([value])

  • value (Number)
  • Returns Manager|Number

设置 timeout , 参数为空则返回值.

#manager.open([callback])

  • callback (Function)
  • Returns Manager

如果 manager初始化时autoConnect 为 false, 这个方法会尝试启用新连接.

callback 可选，当连接成功或者失败时会被调用.

#manager.connect([callback])

同义词： manager.open([callback]).

  • nsp (String)
  • options (Object)
  • Returns Socket

为给定的命名空间(namespace)创建一个Socket

Socket

#socket.id

  • (String)

一个独一无二的会话标识. 当connect 事件触发后被设置, reconnect 触发后更新.

var socket = io('http://localhost');

console.log(socket.id); // undefined

socket.on('connect', function(){
  console.log(socket.id); // 'G5p5...'
});

#socket.open()

  • Returns Socket

开启 socket.

#socket.connect()

同义词：socket.open().

#socket.send([...args][, ack])

  • args
  • ack (Function)
  • Returns Socket

发送(激活)一个 message 事件. 查看 socket.emit(eventName[, ...args][, ack]).

#socket.emit(eventName[, ...args][, ack])

  • eventName (String)
  • args
  • ack (Function)
  • Returns Socket

激活(发送)一个指定的事件，另外的参数也会被一同发送, 所有数据结构都能被支持，包括 Buffer.

socket.emit('hello', 'world');
socket.emit('with-binary', 1, '2', { 3: '4', 5: new Buffer(6) });

ack 是一个可选参数，当客户端响应时会被调用.

socket.emit('ferret', 'tobi', function (data) {
  console.log(data); // data will be 'woot'
});

// server:
//  io.on('connection', function (socket) {
//    socket.on('ferret', function (name, fn) {
//      fn('woot');
//    });
//  });

#socket.on(eventName, callback)

  • eventName (String)
  • callback (Function)
  • Returns Socket

为指定的事件创建一个handler.

socket.on('news', function (data) {
  console.log(data);
});

socket 实际上继承了 每个 Emitter 事件, 类似于 hasListeners, once 或 off (用于移除事件监听).

#socket.compress(value)

  • value (Boolean)
  • Returns Socket

设置一个附加标识给发送(激活)事件，来表明这个事件的数据仅当value为true时会被压缩 compressed.默认 true

socket.compress(false).emit('an event', { some: 'data' });

#socket.close()

  • Returns Socket

手动断开socket连接.

#socket.disconnect()

Synonym of socket.close().
